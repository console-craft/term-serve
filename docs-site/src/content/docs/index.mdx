<section class="ts-home-section ts-home-features" aria-labelledby="features">
	<div class="ts-home-section-heading">
		<span class="ts-home-eyebrow">Features</span>
		<h2 id="features">Batteries-included</h2>
	</div>

	<div class="ts-home-feature-grid">
		<article class="ts-home-feature ts-home-feature-primary">
			<span class="ts-home-feature-tag">PTY over WebSocket</span>
			<h3>Real shell sessions</h3>
			<p>Starts an interactive PTY (bash, zsh, fish, etc.) and streams it over WebSocket.</p>
		</article>
		<article class="ts-home-feature">
			<span class="ts-home-feature-tag">Private Networks</span>
			<h3>Proxy friendly</h3>
			<p>Works behind Caddy, nginx, ngrok, and other reverse proxies via forwarded headers.</p>
		</article>
		<article class="ts-home-feature">
			<span class="ts-home-feature-tag">HTTPS Web Access</span>
			<h3>Cloudflare tunnel</h3>
			<p>Use <code>--tunnel</code> with an installed <code>cloudflared</code> binary for public HTTPS web access.</p>
		</article>
		<article class="ts-home-feature">
			<span class="ts-home-feature-tag">Auth Token</span>
			<h3>Secure by default</h3>
			<p>Binds to <code>127.0.0.1</code> by default and auto-enables token auth when exposed beyond localhost.</p>
		</article>
		<article class="ts-home-feature">
			<span class="ts-home-feature-tag">Command Sandbox</span>
			<h3>Dedicated Commands</h3>
			<p>Serve a specific command on connect (<code>btop</code>, <code>opencode</code>, custom scripts, etc.) instead of a login shell.</p>
		</article>
		<article class="ts-home-feature">
			<span class="ts-home-feature-tag">Modifier Keys</span>
			<h3>Mobile-capable UI</h3>
			<p>Includes touch toolbar keys, sticky Ctrl/Alt, and copy mode tuned for scrollback.</p>
		</article>
		<article class="ts-home-feature">
			<span class="ts-home-feature-tag">Scan QR Code</span>
			<h3>Phone-ready access</h3>
			<p>Prints QR codes for LAN and tunnel URLs so you can open the terminal client from your phone quickly.</p>
		</article>
	</div>
</section>
